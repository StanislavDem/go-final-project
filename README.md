# Итоговый проект курса "Go-разработчик с нуля"

Веб-приложение для планирования задач с аутентификацией и хранением данных в SQLite.

## Функционал
1. Аутентификация через пароль (`TODO_PASSWORD`)
2. Добавление новой карточки задачи с внесением даты, названия, комментария, выбором правила повторений 
3. Хранение данных в SQLite (`scheduler.db`)
4. Редактирование уже добавленных карточек
5. Отметка задач как "Выполненные"
6. Удаление карточек задач
7. Поиск по слову и дате
8. Docker-образ для быстрого запуска

### Переменные окружения
- `TODO_PORT` — порт, на котором запускается сервер (по умолчанию 7540).
- `TODO_DBFILE` — путь к файлу базы данных (по умолчанию `dataBase/scheduler.db`).
- `TODO_PASSWORD` — пароль для аутентификации.

#### Задания со звёздочкой
1. Реализована возможность определять извне порт при запуске сервера. По умолчанию порт берётся из tests/settings.go (tests.Port).
2. Реализована возможность определять путь к файлу базы данных через переменную окружения TODO_DBFILE.
3. Реализовано вычисление следующих дат в случае недель и месяцев. В функции NextDate разбор правил повторения d, y, w, m
4. Реализован функционал поиска задач по дате, первой заглавной букве слова или слову целиком.
5. Реализована аутентификация и docker-образ.

###### Запуск локального сервера
**bash**
`go run ./cmd`

Ссылка на стартовую страницу: http://localhost:7540/login.html
	
###### Инструкция по запуску приложения из репозитория DockerHub

1. Ссылка на репозиторий: https://hub.docker.com/r/fars1r/go-task-scheduler
2. Команда для загрузки образа:
- `docker pull fars1r/go-task-scheduler:v1`
3. Запуск контейнера и монтирование DB через PowerShell: 
- `docker run -d --name go-task-scheduler -p 7540:7540 -v ${PWD}/dataBase/scheduler.db:/dataBase/scheduler.db fars1r/go-task-scheduler:v1`
4. Ссылка на стартовую страницу: http://localhost:7540/login.html
